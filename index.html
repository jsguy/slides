Talk 2: The misojs project: isomorphic JavaScript

Mikkel Bergmann

Misojs is an isomorphic JavaScript framework based on mithril.

Anatomy of a miso app

A miso app is defined using MVC entities, which are complete MVC components by themselves.

This is a somewhat contrived (however valid) MVC entity - defining your app in this manner allows miso to make it isomorphic.

View

The view uses mithril tags to create a virtual DOM element representation of the HTML we want.

Views can return either: a mithril element, a list, or a string.

What about data?

In misojs data is accessed via an API in a uniform manner on both client and server side. Client side we expose the exact same method, using an abstracted transport (JSON RPC2.0).

MVC entity->API: Request API->Server: JSONRPC 2.0 request Note over Server: Server does whatever \nto create the response Server->API: JSONRPC 2.0 Response API->MVC entity: Response

Creating an API

It's very easy to create a new API in miso - all you have to do is create a set of methods that receive a callback function, an error function and an arguments object

Each method must either use the callback or the error function, and can optionally use the args object.
